In this project I am working with a downloaded public dataset from Zomato, a restaurant delivery app based in India. This dataset contains information on over 100 different restaurants around the country and tracks various different factors for each of them, such as:
  • Name of restaurant (as a string)
  • Does the restaurant offer online ordering? (Yes or No)
  • Does the restaurant allow table booking? (Yes or No)
  • Customer rating (out of 5)
  • Total number of ratings
  • Approximate cost of dining for two people (in Rupees)
  • Type of restaurant (one of: buffet, cafe, dining, other)
I will be working in Python using Jupyter Notebook to manipulate and visualize the dataset. I will begin with importing the data as well as doing some preliminary data cleaning and setup to make sure everything is neat and tidy to work with. After that, I will be creating some graphs and visualizations based on the different information and factors that are stored in the dataset. From each of these visualizations, I will be closely inspecting them in order to learn more about the data, find patterns, and draws conclusions.

    Step 2.3: Changing Data Type
    The only areas that I notice any oddities with are the online_order and rate columns. The online_order column contains only two answers: "Yes" and "No", but has a datatype of String. Converting this to a boolean would make it perform much faster when executing searches in this column. However since I am working on a dataset with only ~150 rows, I will keep it as is to avoid writing extra code for my graphs. The rate columns consists of fraction objects represented by a certain value out of 5. This will make graphing this column as a number much more difficult, so I will have to convert these into a decimal value. First I will create a function that converts between these two datatypes, then I will apply it across the entire column.
      In [5]:
      def frac2float(frac):
    """
    Converts a fraction object into a float
    Parameters:
        frac (object): a fraction, whose string looks like 'X/Y'
    Returns:
        float: the decimal value of the fraction
    """
    # make the fraction into a string:
    f = str(frac)
    # split the string around the "/" symbol:
    f = f.split("/")
    # convert each part to floats take only the first
    f = float(f[0])
    return f
# Now apply the function to the rates column and see that rates are now floats:
df['Rating'] = df['Rating'].apply(frac2float)
print(df['Rating'].loc[0:4])

        Graph 2: Rating vs Rating Count
        This graph is a scatterplot displaying the relation between Rating and Rating count in order to determine any possible correlation between the two. It also includes coloring for the top and bottom 10% of restaurants by Approximate Two Person Cost. I took the logarithm of Rating Count in order to make the graph more readable since this factor was very skewed towards higher counts.
        First off I see that most ratings lie within the range of 3.5 to 4, and most log counts lie from 3 to 6 (20 to 400 actual count). The relation between rating count and rating doesn't seem to be entirely linear, but rather positively quadratic. There isn't enough data to say for sure, but it seems that restaurants with much higher or much lower ratings tend to have a higher count of ratings than restaurants with average ratings. This would make sense since a particularly good or bad experience with a restaurant could make a customer more likely to leave a review than an average experience would. Looking at the extremes of the Approximate Two Person Cost I don't seem to notice any particular relation with Rating, however there may be some vague positive correlation with rating count. Generally, it seems that cost doesn't have a very meaningful effect on either of these factors.
